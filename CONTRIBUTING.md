# Contributing to DeVana

Thank you for your interest in contributing to DeVana! We welcome contributions from the structural dynamics, vibration control, and optimization communities.

## How to Contribute

### 1. Reporting Bugs
- Check the [Issue Tracker](https://github.com/mahan2079/DeVana/issues) to see if the bug has already been reported.
- If not, open a new issue. Include a clear description of the problem, steps to reproduce it, and your environment (OS, Python version, dependencies).

### 2. Suggesting Enhancements
- We are always looking for new optimization algorithms, physics models, or UI improvements.
- Please open an issue labeled `enhancement` to discuss your idea before starting implementation.

### 3. Pull Request Process
1. Fork the repository.
2. Create a new branch for your feature or fix (`git checkout -b feature/AmazingFeature`).
3. Ensure your code follows the existing style (PEP 8 for Python).
4. **Write tests** for any new functionality in the `tests/` directory.
5. Run the existing tests to ensure no regressions:
   ```bash
   python tests/run_all.py
   ```
6. Commit your changes with descriptive messages.
7. Push to your fork and submit a Pull Request.

## Code of Conduct
By participating in this project, you agree to abide by our code of conduct. We strive to maintain a welcoming and inclusive environment for all contributors.

## Areas for Contribution
- **New Optimization Workers**: Implementation of hybrid metaheuristics or specialized RL agents.
- **Continuous Systems**: Helping with the `Continuous_beam` module and FEM integration.
- **Visualization**: Adding new plot types or improving the results dashboard.
- **Documentation**: Improving tutorials, adding examples, or refining mathematical explanations.
