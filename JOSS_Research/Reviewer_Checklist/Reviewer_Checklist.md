# JOSS Reviewer Checklist

This checklist is used by reviewers during the GitHub-based review process.

## Conflict of Interest & Code of Conduct
- [ ] I confirm that I have read the JOSS Conflict of Interest Policy and have no conflicts with this submission.
- [ ] I confirm that I will adhere to the JOSS Code of Conduct.

## General Checks
- [ ] **Repository:** Is the source code for this software available in a public repository?
- [ ] **License:** Does the repository contain an OSI-approved license?
- [ ] **Version:** Does the release version match the submission?
- [ ] **Authorship:** Is the submitter a major contributor?

## Functionality
- [ ] **Installation:** Does the software install as documented? (Check dependencies).
- [ ] **Functionality:** Do the core claims about what the software does hold up?
- [ ] **Performance:** Are performance claims (if any) verified?

## Documentation
- [ ] **A Statement of Need:** Is the purpose of the software clear?
- [ ] **Installation Instructions:** Are there clear instructions for installing the software?
- [ ] **Example Usage:** Are there examples (e.g., tutorials, notebooks, or scripts)?
- [ ] **API Documentation:** Is the core functionality well-documented?
- [ ] **Automated Tests:** Are there tests to verify correctness? Do they pass?
- [ ] **Community Guidelines:** Are there clear instructions for contributing, reporting bugs, and seeking support?

## Paper Content
- [ ] **Summary:** Is the summary accessible to a non-specialist?
- [ ] **Statement of Need:** Does it clearly explain the research problem solved?
- [ ] **State of the Field:** Is there a proper comparison to related tools?
- [ ] **References:** Do the references have DOIs?
- [ ] **AI Disclosure:** If AI was used, is it properly disclosed?
