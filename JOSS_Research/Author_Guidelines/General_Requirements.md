# JOSS General Requirements for Authors

To be eligible for publication in the Journal of Open Source Software (JOSS), the software must meet the following criteria:

## 1. Open Source License
The software must have an **OSI-approved open source license** (e.g., MIT, Apache 2.0, GPL v3). A `LICENSE` file must be present in the repository root.

## 2. Research Application
The software must have a clear use case in research. JOSS is for "research software" — software that is used in the process of creating research results (modeling, data analysis, instrument control, etc.).

## 3. Significant Contribution
The software must represent a "non-trivial" piece of work. It should enable new research or significantly improve existing workflows. 
- Avoid "one-off" scripts.
- Should have a clear target audience of researchers.

## 4. Maturity
- The software must be **feature-complete** and ready for use.
- It must have at least **6 months of public development history** (as of 2026 guidelines).
- It should have a versioned release (e.g., v1.0.0).

## 5. Submitter Status
The person submitting the paper must be a **major contributor** to the software.

## 6. Repository Requirements
- Must be hosted on a public platform (GitHub, GitLab, Bitbucket).
- Must have a clear README.
- Must include contribution guidelines (`CONTRIBUTING.md`).
- Must have an issue tracker.
- Must include automated tests.
